# KRONIA NURSE — KNOWLEDGE CENTER (ARCHITECTURE CONTEXT)

## MISSÃO

A tela NÃO representa uma biblioteca de documentos.

Ela representa um CENTRO DE CONHECIMENTO CLÍNICO.

Toda a experiência deve transmitir que o aplicativo possui uma Base de Conhecimento Inteligente construída sobre evidências clínicas.

O usuário nunca deve sentir que está navegando por PDFs ou arquivos.

Ele deve sentir que está navegando por conhecimento estruturado.

## O QUE É UM KNOWLEDGE OBJECT

Cada item exibido representa um objeto de conhecimento.

Exemplo:

Punção Venosa Periférica

Esse objeto reúne:

- Resumo clínico
- Indicações
- Contraindicações
- Materiais
- Preparo
- Passo a passo
- Cuidados
- Complicações
- Alertas
- Checklist
- Perguntas frequentes
- Atualizações
- Evidências
- Referências

O usuário nunca enxerga isso como diversos documentos.

Ele enxerga como um único conhecimento.

## PRINCÍPIO FUNDAMENTAL

O KRONIA Nurse não é uma biblioteca.

O KRONIA Nurse é uma Plataforma de Conhecimento Clínico.

Toda decisão de UX, UI, arquitetura da informação e organização visual deve reforçar essa percepção.

Se existir dúvida entre "mostrar um documento" ou "mostrar conhecimento", sempre escolher conhecimento.

Essa é a identidade central da plataforma.
